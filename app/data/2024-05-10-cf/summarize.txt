En este video, comparto cómo desplegar una aplicación Angular en Cloudflare, destacando las ventajas de este proveedor en términos de innovación y herramientas de IA disponibles. Con un enfoque en aplicaciones de una sola página (Single Page Application - SPA), presento un proyecto de ejemplo: una sencilla aplicación de e-commerce que interactúa con una API. A lo largo del video, guío paso a paso el proceso de despliegue en Cloudflare, desde la creación de una cuenta hasta la configuración y conexión con GitHub para automatizar el despliegue mediante commits.

Cloudflare se destaca por sus servicios de inteligencia artificial, protección contra ataques, hosting, y gestión de dominios. Exploro dos características principales de Cloudflare: Workers y Pages, resaltando la utilidad de Workers para APIs por su funcionamiento en el Edge y la simplicidad de Pages para desplegar aplicaciones estáticas. El proceso de despliegue en Cloudflare Pages involucra conectar el proyecto de GitHub, seleccionar el framework, y ajustar la configuración de compilación según el builder utilizado en Angular.

Además, realizo un cambio en el proyecto para demostrar cómo Cloudflare actualiza automáticamente la aplicación con cada commit en GitHub. También cubro la configuración de un dominio personalizado en Cloudflare, mostrando la facilidad de gestionar y proteger dominios dentro de esta plataforma.

Finalmente, menciono planes futuros para explorar el Server Side Rendering (SSR) en Angular para mejorar el SEO y la performance del e-commerce, anticipando contenido sobre cómo habilitar SSR en Cloudflare.
