En este video, te enseño cómo desplegar una aplicación con Server Side Rendering (SSR) en Cloudflare, destacando la importancia de esta técnica para aplicaciones Angular que ya no se basan únicamente en archivos estáticos. Exploro otras opciones como Heroku pero me centro en Cloudflare por sus características únicas de Edge Computing, que es un entorno reducido de Node, ideal para ejecutar JavaScript del lado del servidor de manera más eficiente y económica.

Durante la explicación, destaco cómo Cloudflare puede manejar tanto archivos estáticos como aplicaciones dinámicas con SSR, utilizando un comando específico que configura un proyecto Angular desde cero para este entorno. Aunque el comando es útil, se necesita adaptar aplicaciones existentes manualmente, lo que demuestro al copiar y modificar configuraciones del proyecto generado automáticamente por Cloudflare en mi aplicación existente.

Después de ajustar configuraciones y instalar dependencias necesarias, procedo a demostrar la implementación y pruebas locales antes de realizar el despliegue final en producción. Este despliegue se integra con GitHub para facilitar actualizaciones automáticas y asegurar que la aplicación se distribuya correctamente a través de Cloudflare Pages.

Finalmente, discuto las ventajas significativas de usar Cloudflare para SSR en Angular, como la reducción de costos, mejoras en latencia y la simplificación del despliegue. Además, invito a la audiencia a comentar y compartir sus experiencias o preferencias entre las diversas opciones de nubes y tecnologías disponibles para el despliegue de aplicaciones modernas.
